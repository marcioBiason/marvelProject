Modularização

No Angular podemos criar modulos na construção de nosso projeto, ou seja, dividir várias partes do sistema em partes menores que possuem tarefas específicas. O uso deste recurso traz muitas vantagens como a:

- Reutilização do código

- A facilidade de manutenção

- Melhor legibilidade e compreensão do código

Portanto, o conceito de dividir o seu código em blocos que se comunicam entre si e que cada parte tenha uma responsabilidade especifica pode ser chamado de modularização.

Component factory

A idéia é criar componentes que são reutilizados em mais de um módulo do projeto. Sendo que uma tela pode conter vários componentes e quanto mais dinâmicos e reutilizáveis sejam esses componentes, maior agilidade você ganha ao reutilizá-los.

Component styled

A estilização dos componentes serve para criar identidade ao seu projeto, você pode definir o estilo de um componente na própria criação, padronizando os componentes, além de poder criar mais de um tema de estilo para o seu componente.

Gerenciamento de estado

Serve para a comunicação entre componentes, você pode observar ações em outros componentes. Caso uma nova ação seja disparada o componente pode adotar determinado comportamento baseado na ação disparada.

Clean code

Se refere a um conjunto de boas práticas que você deve aplicar para obter uma maior legibilidade e manutenibilidade do seu código.

O código limpo é eficiente, simples, deve ter minimas dependências, sem linhas duplicadas, fácil de corrigir, ter padrões definidos, de fácil leitura e entendimento, e por ultimo ter cobertura de testes.
